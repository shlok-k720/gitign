gitign

Add Git ignore rules, untrack matching files, and safely choose whether local files stay, are backed up, or are removed.

BashGitZsh / Bash / Fish

What it does

Plans first

Shows each resolved pattern and candidate match count before changes are made. Use --dry-run for a no-change preview.

Updates Git safely

Adds rules, untracks matching index entries, and commits by default only when the staging area starts clean.

Handles local files explicitly

Keeps local files by default. Opt into permanent deletion, native operating-system Trash, or recoverable backups.

Install

Clone the repository anywhere you can edit it. On macOS, ~/Library/Application Support/gitign is recommended, but it is not required.

$ git clone https://github.com/shlok-k720/gitign.git ~/gitign
$ bash ~/gitign/compile-gitign.sh
$ source ~/.zshrc
$ gitign --version

The installer creates ~/.local/bin/gitign, injects the release version, and configures the PATH block for the detected Zsh, Bash, or Fish shell. Override detection with --shell zsh, --shell bash, or --shell fish. Gitign supports macOS, Linux, Windows Git Bash, and WSL.

$ bash compile-gitign.sh --print-install-path
$ git pull && bash compile-gitign.sh --reinstall
$ bash compile-gitign.sh --uninstall

Safety and local file choices

CommandIgnore ruleGit indexLocal result
gitign build/addedmatching paths untrackedkept
gitign --delete_local build/addedmatching paths untrackedpermanently deleted
gitign --trash build/addedmatching paths untrackedmoved to the native operating-system Trash
gitign --backup-dir ../backup build/addedmatching paths untrackedmoved to backup directory
 
$ gitign --dry-run --verbose --delete_local build/
# Preview patterns, tracked entries, potential local matches, and commit behavior.

$ gitign --backup-dir ../gitign-backups '**/*.log'
$ gitign --trash nodemodules
Deletion is opt-in. Normal gitign commands never delete local files. --delete_local permanently removes precisely matched paths. --trash automatically uses macOS Trash, the FreeDesktop Linux Trash (including WSL), or the Windows Recycle Bin in Git Bash. Use --backup-dir when recovery matters.
PlatformTrash adapter
macOSUser Trash at ~/.Trash
Linux / WSLFreeDesktop Trash at $XDG_DATA_HOME/Trash or ~/.local/share/Trash
Windows Git BashWindows Recycle Bin through PowerShell

Linux selects a same-filesystem FreeDesktop Trash directory when needed. WSL uses that Linux adapter because Windows cannot recycle WSL UNC paths. Unsupported Bash environments, or Git Bash without PowerShell, show an error instead of falling back to permanent deletion. --backup-dir is the portable recovery option.

Before local handling, gitign asks interactive users for confirmation. It also confirms automatic commits for broad operations (globs or multiple patterns). In non-interactive scripts and CI, it proceeds automatically; use --dry-run to audit a command or --yes to state approval explicitly.

Gitign evaluates deletion with Git's ignore matcher after adding the requested rule. This prevents an overlapping older ignore rule from causing unrelated local paths to be removed.

Automatic commits and output

$ gitign --commit-message "Ignore generated assets" dist/
$ gitign --no-auto-commit --delete_local build/
$ gitign --quiet logs

Auto-commit is enabled by default. It requires a clean staging area, generates a focused commit, and explicitly reports when no commit is needed. Use --no-auto-commit to review changes manually. --verbose lists every planned path; --quiet hides routine output while retaining errors. Terminal output is colorized when supported.

Repository configuration and global rules

Put a .gitignrc file at the repository root to set reusable defaults; command-line options win.

# .gitignrc
auto_commit=false
recursive_filenames=true
deletion_mode=backup
backup_dir=../gitign-backups
confirm=true

Supported keys: auto_commit, delete_local, deletion_mode, backup_dir, recursive_filenames, global_ignore, confirm, verbose, quiet, and commit_message.

$ gitign --global env
# Writes to Git's configured global excludes file, creating ~/.config/git/ignore if needed.

Undo

$ gitign --backup-dir ../gitign-backups '**/*.log'
$ gitign --dry-run --undo
$ gitign --undo

Gitign stores one transaction under .git/gitign/. Undo removes newly added rules, restores index entries, and restores backup-mode local files. An automatic commit is undone with a revert only while that commit remains HEAD. Permanently deleted and trashed files cannot be restored automatically.

How the command works

1. Resolve
Find the repository, load config, and expand aliases.
2. Preview
Show resolved rules and matching path counts.
3. Apply
Add ignore rules, untrack paths, and handle local matches if requested.
4. Record
Save undo metadata and optionally make a focused commit.
