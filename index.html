gitign

Add Git ignore patterns and stop tracking matching files without deleting the files from your computer.

BashGitmacOS / Zsh

What it does

Writes patterns

Adds each supplied pattern to the repository root’s .gitignore, without adding duplicates.

Untracks and commits

Runs git rm --cached for matching tracked files, then commits by default. Files remain on disk.

Finds the repository

Works from subfolders and asks for a repository path or init when needed.

Install

Clone the repository into any editable directory, then run its installer. On macOS, keeping it in ~/Library/Application Support/gitign is recommended, but not required.

$ git clone https://github.com/shlok-k720/gitign.git ~/gitign
$ bash ~/gitign/compile-gitign.sh
$ source ~/.zshrc
# gitign is now available on your PATH

The installer makes ~/.local/bin/gitign point to the editable gitignore.sh source and adds ~/.local/bin to PATH in ~/.zshrc. Pulling updates in the clone updates the command.

Quick start

Run gitign from any directory within a Git working tree.

$ gitign dsstore
# Adds **/.DS_Store and untracks matching .DS_Store files

$ gitign nodemodules
# Adds **/node_modules/ and untracks tracked dependencies

$ gitign database.db
$ gitign '*/tmp.js'
$ gitign build/

Presets and patterns

CommandPattern addedMeaning
gitign dsstore**/.DS_StoreIgnore macOS metadata files recursively below the current directory.
gitign nodemodules**/node_modules/Ignore all node_modules directories recursively below the current directory.
gitign cache/cache/Ignore a directory and its contents.
gitign '**/*.log'**/*.logIgnore matching files recursively.
Quote glob patterns. In Zsh, quote arguments containing *, ?, [, or ! so the shell does not expand them before gitign receives them.

Patterns are relative to the directory where you run gitign. A pattern beginning with / is relative to the repository root instead.

Automatic commits

gitign commits its own changes by default, using gitign: ignore <pattern> for one pattern or gitign: ignore N patterns for several.

$ gitign database.db
# Adds the pattern, untracks matching files, and creates a commit.

$ gitign --no-auto-commit database.db
# Makes the same changes but leaves them for your review.

--auto-commit explicitly enables the default. To avoid capturing unrelated work, auto-commit stops if the staging area already has changes.

When setup is missing

$ gitign app.log
No Git repository found. Enter a repository path, or type init to initialize one in /your/folder:
init

No .gitignore found. Enter a path inside this repository, or type init to create /your/folder/.gitignore:
init

Outside a repository, enter the path to an existing Git working tree, or enter init to run git init in the current directory. If the chosen repository has no root .gitignore, enter any path inside it or init to create the root ignore file.

How it works

1. Resolve location
Git finds the working tree and repository root.
→
2. Normalize pattern
Presets expand and subfolder paths are scoped.
→
3. Update Git
Append to .gitignore, remove matching index entries, then commit.

The central logic is intentionally small:

# Expand friendly preset names into Git ignore patterns.
case "$argument" in
  dsstore)     pattern="**/.DS_Store" ;;
  nodemodules) pattern="**/node_modules/" ;;
  *)             pattern="$argument" ;;
esac

# Keep matching files locally but remove them from Git's index.
git rm --cached -r --ignore-unmatch -- "${tracked_paths[@]}"

Reviewing instead of committing

Pass --no-auto-commit when you want to inspect and commit the result yourself.

$ gitign --no-auto-commit '**/*.log'
$ git status
$ git add .gitignore
$ git add -u
$ git commit -m "Ignore local generated files"
